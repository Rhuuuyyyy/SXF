# `docs/` — External-facing Documentation

## Context & Purpose

`docs/` is the **public-facing documentation surface** distinct from the
Obsidian vault under [[000_Home_Backend_MOC]]:

| Surface | Audience | Format | Source |
|---|---|---|---|
| **Obsidian vault** (`Notes/`) | Internal team, AI agent | Markdown w/ wikilinks | hand-curated |
| **`docs/`** | Frontend team, partners, auditors | MkDocs / static site | partly autogen |
| **OpenAPI** | Frontend team, integrators | JSON 3.1 | autogenerated by [[FastAPI]] |

The vault is the "why"; `docs/` is the "how to consume". They reference
each other but have different authorship rules:

- The vault is the **source of truth** under [[000_AI_OBSIDIAN_PROTOCOL]].
- `docs/` may **extract and publish** vault excerpts, but never
  contradicts them.

## Children (canonical layout)

```
docs/
├── index.md                # landing page
├── architecture.md         # cleaned export of [[001_Architecture_and_Context]]
├── api/
│   ├── overview.md
│   └── openapi.json        # build-time copy from /api/v1/openapi.json
├── runbooks/
│   ├── local-dev.md
│   └── deploy.md
└── compliance/
    ├── lgpd.md             # exec summary of [[006_LGPD_PII_Strategy]]
    └── audit.md            # exec summary of [[007_Audit_Logging_Middleware]]
```

## Allowed dependencies
- Markdown / MkDocs / mkdocs-material themes.
- Build script that copies `/api/v1/openapi.json` from a running app
  into `docs/api/openapi.json`.

## Forbidden imports
- `docs/` is **content**, not code. Nothing in `docs/` should be
  imported by anything in [[dir_app]].
- Do **not** include real PII or production secrets in any example —
  see [[006_LGPD_PII_Strategy]].

## Patterns
- **Single-source-of-truth wins.** When a vault note and a docs page
  drift, the vault note is canonical and the docs page is regenerated.
- **Public diagrams are rendered.** Mermaid sources stay in the vault;
  docs ship rendered SVGs to remain readable without Obsidian.
- **Versioned docs** mirror the API version: `/v1/`, `/v2/`. Out-of-
  support versions move under `docs/archive/`.
